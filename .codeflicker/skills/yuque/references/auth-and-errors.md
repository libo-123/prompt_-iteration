# 认证与错误

当语雀调用失败或准备新请求时使用本文件。

## 认证

- 基址：`https://www.yuque.com`
- 请求头认证：`X-Auth-Token: <token>`
- OpenAPI 文件中的安全方案名称：`authToken`
- 本地认证文件：从 `.env.example` 复制得到的 `skills/yuque/.env`

若存在 `skills/yuque/.env`，脚本会自动加载。`.env` 中的值仅填充缺失的环境变量，不会覆盖 Shell 中已设置的变量。

实际生效的令牌解析顺序：

1. `--token`
2. `YUQUE_TOKEN`
3. `YUQUE_AUTH_TOKEN`

若尚未接入认证，缺少 `X-Auth-Token` 应视为首要失败原因之一。

## 常见失败模式

### 400

请求形态无效。

检查：

- 路径参数格式是否正确
- 查询参数是否使用文档中的名称
- 枚举值是否与 OpenAPI 文件一致
- 数字字段是否被随意传成字符串

### 401

令牌缺失、无效或权限范围不足。

检查：

- 请求头名称是否恰好为 `X-Auth-Token`
- 令牌是否仍有效
- 令牌是否能访问目标 API 域
- 本地是否存在 `skills/yuque/.env` 且为真实令牌，而非 `.env.example` 中的占位符

### 403

令牌有效但无法对目标资源执行操作。

常见原因：

- 在无编辑权限的知识库中尝试写入
- 尝试读取令牌范围外的私有内容
- 无管理员权限却尝试修改团队成员

### 404

未找到目标资源。

常见原因：

- `group_login` 错误
- `book_slug` 错误
- `book_id` 错误
- `doc_id` 错误
- 目录节点 UUID 来自过期状态

### 422

请求已通过路由但校验失败。

常见原因：

- 创建文档时缺少 `body`
- 不支持的 `format`
- 无效的目录动作或缺少必填动作字段
- 可见性取值格式错误

### 429

请求频率过高。

应对方式：

- 减少并发扇出
- 避免在同一步骤内重复读取
- 谨慎重试，避免疯狂循环

### 500

上游接口故障。

在重试写入前：

- 若可能，重新读取资源
- 确认前一次写入是否部分成功
- 避免盲目重放大范围目录或批量变更

## 请求构造默认值

- 文档格式默认 `markdown`
- 写入载荷保持明确
- 除非用户要求修改，否则保留既有 `slug` 与 `public`
- 任何目录写入前先读取当前目录
