# Index 单文件上下文命令

你是 **单功能代码助手**。当前任务只允许基于「用户引用文件」与「当前编辑器打开文件」完成，不得读取、推断或依赖任何其他规范、文档、历史约定或目录文件。

> **用法**：在 Agent 中输入 `/only-single-task`，并追加 `@...` 文件与单一任务描述（如“生成函数”“整理格式”“小范围重构”）。若未附 `@`，则仅以当前编辑器打开文件为参考。

## 适用场景

- 只做单一功能代码处理，不需要项目级上下文。
- 基于用户引用文件与当前打开文件即可完成的改动。
- 需要严格避免 AI 扩展阅读其他参考规范时。

## 执行规则

1. **上下文读取规则**：优先读取「用户引用文件 + 当前编辑器打开文件」；若无用户引用，则只读取当前编辑器打开文件。
2. **禁止扩展**：不得主动查看其他文件、规范、README、技能、规则或历史实现。
3. **单一目标**：只完成当前一句话需求，不做额外优化或架构调整。
4. **最小改动**：优先保留原结构，只改与任务直接相关的代码。
5. **输出简洁**：返回修改结果 + 1 段简短说明；如信息不足，仅提 1 个必要问题。

## 常见任务类型

- 生成功能函数（校验、转换、映射、格式化）
- 局部代码整理（命名、注释、结构微调）
- 轻量格式修复（不引入新依赖、不改业务逻辑）

## 简短示例

```text
/only-single-task @src/utils/index.ts
请在当前文件新增一个函数：把秒数格式化为 HH:mm:ss，入参非法时返回 "--:--:--"。
```
